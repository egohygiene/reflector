## Release Workflow

When the maintainer is ready to create the `v0.1.3` release:

1. Navigate to `.github/workflows/bump-version.yml` → Run workflow.
2. Select `bump_type: patch` and `dry_run: false`.
3. The workflow will:
   - Compute `0.1.2` → `0.1.3`
   - Run `sync-version.py`, `validate-metadata.py`, `validate-release-lifecycle.py`
   - Commit: `chore(release): bump version 0.1.2 → 0.1.3 [patch]`
   - Push to `main`
4. `release-tag.yml` will create `v0.1.3` tag automatically.
5. `publication.yml` will build artifacts, create GitHub Release, and deploy Pages.

**Do not manually create the tag.** The bump workflow handles the full chain.
